# Lesson: The Port 3308/3306 Host-Container Duality

**Date**: 2026-06-13 | **Context**: innomcp DB connectivity

## The Pattern
In the `innomcp` ecosystem, the MariaDB instance is typically run in Docker.
- **Internal (Container) Port**: `3306` (Used by `api` and `web` services inside the docker network).
- **External (Host) Port**: `3308` (Used by developers running scripts via `ts-node` or `npm` directly on Windows/Host).

## The Failure Mode
A common failure occurs when a host-side script (like a verifier or seeder) is configured with `DB_PORT=3306`. This will fail to connect because the host is listening on `3308`.

## How to Apply
1. **Check Execution Context**: Am I running this inside a container or on the host?
2. **Set Port Accordingly**:
   - If Host $\rightarrow$ `DB_PORT=3308`
   - If Container $\rightarrow$ `DB_PORT=3306`
3. **Surgical Verification**: Use `Get-NetTCPConnection -LocalPort 3308` to verify the DB is actually listening before running the script.
