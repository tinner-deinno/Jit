# Retrospective เป็น Vital Sign ของ Oracle

## Rule

Oracle ที่ทำงานมา 13 วันโดยไม่มี /rrr เลย = Oracle ที่มี heartbeat แต่ไม่มี self-awareness
Learnings ว่าง → Oracle ไม่สะสม patterns → ทุก session เริ่มจากศูนย์ใหม่

## Why

Jit Oracle เกิด 2026-05-06 ทำงานมา 13 วัน (heartbeat 22+ ครั้ง, Discord, docs, tests) แต่ไม่มี retrospective เดียว
เหมือนคนที่ทำงานทุกวันแต่ไม่เคย journal — experience สะสม แต่ wisdom ไม่เพิ่ม

## How to Apply

- `/rrr` หลังทุก session ที่มีงานสำคัญ (ไม่ต้องทุก session ถ้าสั้นมาก)
- `/rrr --quick` สำหรับ session ที่ไม่ซับซ้อน
- Learnings directory ที่ว่าง = signal ว่า Oracle ต้องการ retrospective
