## Key Discoveries (Visual QA)

1. **Stale Turbopack cache bug**: port 3000 served old bundle. `146a.js` (MDESBrandHeader) was NOT loaded. After killing port 3000 and starting on 3001: ✅ totalHeaders=2, MDESBrandHeader visible at top=0

2. **CSS sticky broken by overflow-x-hidden**: layout.tsx `<main overflow-x-hidden>` silently breaks all `position: sticky` children. Removing it fixed MDESBrandHeader sticking to viewport top.

3. **Page was 2000px tall**: Old hero text + new ChatWelcomeHero both rendering. Fixed by hiding old block with `class="hidden"`.

4. **DB port mismatch**: MariaDB container on Docker port 3306 (internal), backend .env said 3308. Fixed → login works via JWT mock.

## Honest Feedback (3 friction points)

1. **Token generation ≠ completion**: Burning 917k tokens produced a LOT of components, many of which are orphaned (not imported, not wired). The advisor called this correctly. Next time: generate+verify loop, not generate+generate+generate. Every 3 swarm waves should have 1 visual check.

2. **Stale cache caused 3+ hours of confusion**: The dev server on port 3000 was serving old code. If I'd killed it at the start, we'd have seen MDESBrandHeader working much earlier. Lesson: always verify dev server is fresh. Add to pre-development checklist: kill all node processes, `rm -rf .next`, fresh start.

3. **DB was down the entire time and nobody noticed until login QA**: Backend health showed DB unhealthy since the beginning, but all tests passed because pre-commit tests don't require DB. A real health check at session start would have caught this. Added `.env.example` with DB config but never verified the actual DB connection.

## Lessons Learned

1. **Visual QA is non-negotiable** — `tsc --noEmit PASS` + `smoke PASS` ≠ "works". The app must be run visually after every significant UI change.

2. **Playwright headless = perfect for autonomous visual verification** — `getBoundingClientRect()`, screenshot crops, and route mocking let me do proper QA without human intervention. This should be standard practice.

3. **CSS sticky + overflow-x-hidden**: Any ancestor `overflow: hidden` (even one axis) breaks `position: sticky` descendants. Known CSS quirk, easy to miss.

4. **Turbopack stale cache**: In Next.js dev, if a file was modified after server start, HMR should update it. But if the server never compiled the latest version (was started before the edit), a full restart + `.next` delete is needed.
