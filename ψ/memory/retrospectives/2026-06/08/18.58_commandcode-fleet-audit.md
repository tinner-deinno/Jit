## Lessons Learned

1. **Verify dispatch participation ก่อนประกาศ readiness** — agent definition file ที่มีอยู่ไม่ได้หมายความว่า agent participate ใน actual dispatch loop รัน `mother.js status` หรือดู fleet log ก่อนพูดว่า "พร้อม N%"

2. **Provider หายไปจาก fleet ไม่ใช่แค่ DOWN** — provider อาจ ALIVE แต่ไม่ถูก dispatch เพราะ BACKEND_ORDER ไม่รวม หรือ prioritization ต่ำเกินไป ต้อง trace จาก backend order → probe → actual dispatch log

3. **commandcode เป็น multi-model hub ไม่ใช่แค่ single provider** — 28 models รวม Claude Pro tier ใน budget แยก — ออกแบบ routing ให้ใช้ประโยชน์นี้: Claude models สำหรับ reasoning-heavy, DeepSeek-flash สำหรับ batch throughput
