## Lessons Learned

1. **Return type changes unlock entire pipelines** ŌĆö changing `synthesizeResults: Promise<string>` to `Promise<{text, winnerId}>` was the linchpin for all win tracking. When a function's output needs to carry more information, the right fix is the return type, not a side-effecting global.

2. **System reminders are the most reliable state source after sub-agent failure** ŌĆö when agents fail mid-task, the system reminders showing file modifications contain the ground truth of what actually happened. Cross-reference reminders before re-doing work.

3. **Always-on providers (no key required) are essential for CI** ŌĆö having 3 always-on providers (ollama-local, innova-bot, innova-oracle) means the mother dispatch always fires at least 3 agents even in a keyless CI environment. Design systems with "zero-config baseline" that degrades gracefully.

4. **Composite score formulas need cold-start safety** ŌĆö historical-stats-based scores fail on the first dispatch. Using in-run `1/(1+latencyMs/1000)` instead of stored averages gives meaningful ordering from iteration 1.
