## Honest Feedback
1. **Slow cold-start providers wrecked iteration speed.** ollama_mdes takes ~28s cold; every live phase was ~140s, and the first probe's 25s timeout falsely marked it dead. I had to re-probe at 35s and re-reason. Verifying a "dead" verdict with a real call should have been my first instinct, not my third.
2. **Provider quota is a consumable I treated as free.** Repeated phase runs + a model-mismatch retry storm exhausted the ollama_cloud weekly limit mid-session, permanently shrinking the usable fleet for the rest of the work.
3. **Over-checkpointing against an autonomous stop-hook.** I called AskUserQuestion / wrote full "should I continue?" checkpoints several times while the hook was clearly driving continuous autonomous work — burned turns re-asking what was already answered.

## Lessons Learned
- **A probe timeout shorter than a service's cold-start will lie to you.** Before trusting "provider dead," confirm with one real call at a generous timeout — cold-start ≠ down.
- **Backend and model are coupled — never override one without the other.** Routing a request to a new backend with the old model causes 404s *and* can burn quota via the retry storm that follows. A partial fix can damage scarce shared resources.
- **Diverse-lens verify swarms find what single-pass review misses.** 3–5 cheap agents each attacking a distinct surface (injection, scale, concurrency, corruption, fidelity) reliably surfaced real bugs; the cost is trivial vs the bugs caught.
- **Neutral defaults must not outrank measured-but-imperfect values.** A "no data = 1.0" default let untested options beat proven-99% ones; neutral belongs at the *midpoint* (0.5), not the max.
