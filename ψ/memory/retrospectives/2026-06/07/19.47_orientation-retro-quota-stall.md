## Lessons Learned

1. **Don't spawn a subagent for a single deterministic command.** A subagent
   costs quota, can fail wholesale (rate/usage limits), and blocks the parent
   workflow; an inline call fails visibly and retries cheaply. Reserve
   subagents for work that genuinely needs parallel context.
2. **Report-time state beats session-start state.** In a repo shared by
   parallel sessions/loops, re-run `git status`/`git log` immediately before
   writing any report or pending table — snapshots in system context or
   earlier tool output go stale within hours.
3. **On Windows oracles, assume no python.** Skill scripts embedding
   `python3 -c` silently hit the Store-alias trap; ship node equivalents
   (this repo already migrated limbs to Node.js in `c2f8942` for the same
   reason — the pattern is confirmed fleet-wide).
