## Lessons Learned

1. **Always verify file state before describing what "should" have happened** — when sub-agents fail, the system reminders show what actually got written. Cross-reference those before drafting commit messages or "done" claims.

2. **Autonomous loops need human-readable status files** — the mother loop writes to ψ/outbox per cycle, which makes it auditable across sessions. Any long-running autonomous process should write to a persistent log that survives session resets.

3. **Rate limit exhaustion is a systemic signal, not a bug** — when 5+ agents hit limits simultaneously, it means you're operating at the edge of the rate budget. Design for this: prefer sequential over parallel when near limits.
