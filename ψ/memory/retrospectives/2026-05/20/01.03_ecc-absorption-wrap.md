## Honest Feedback

**Three friction points from this session:**

1. **ψ path not in Oracle root detection.** The skill's bash detection logic (`git rev-parse` + check for CLAUDE.md + ψ/) is designed for Unix oracle repos. In this Windows setup, the Oracle is not in `pwd` — it's in `C:\Users\admin\Jit`. The detection failed silently to "use pwd" and I started creating directories in the wrong place. The memory cross-ref saved the session, but only because I happened to read the ecc_integration.md file. A more robust approach would be: check memory for ψ location *first*, before running the Oracle root detection bash script.

2. **Windows/bash path friction.** The detection uses `git rev-parse` and bash path encoding (`sed 's|^/|-|...'`), which behaves differently on Windows-hosted git. The encoded project path `C--Users-admin` works but is fragile — if the user changes drive or username, encoding breaks. This is a structural issue with the harness, not something fixable in this session.

3. **Background agent returned no snippet content.** The timestamp miner's output confirmed timestamps but no message snippets (all filtered by the `<command-name>` check). This is correct behavior for a skill-only session, but it means the Timeline section is reconstructed purely from AI memory — which in a compressed context is limited. The miner design works for real sessions; for pure /rrr sessions, it will always return empty snippets.

## Lessons Learned

1. **Read memory for environment context before running detection scripts.** When an Oracle's ψ is not co-located with the working directory (common in Windows setups where home ≠ repo), the standard bash detection will fail to find it. Always check memory for stored ψ / Oracle root paths before running detection.

2. **Meta-sessions (pure /rrr) still need retros, even if brief.** The discipline of writing a retro even for a 5-minute wrap session preserves session continuity and closes the metrics row. Skipping it creates gaps in pattern detection.

3. **Capability installs should be followed by a single activation check.** After a large absorption (72+ artifacts), a quick `/recap` or skill inventory check at the start of the next session confirms what's live before reaching for new tools.
